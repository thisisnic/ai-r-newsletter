# AI Tooling & R Ecosystem Weekly Digest

A quiet-but-productive week dominated by incremental releases and the usual ambient hand-wringing about agentic coding.

---

## R Packages

**[ellmer 0.4.1](https://github.com/tidyverse/ellmer/blob/HEAD/NEWS.md#ellmer-0-4-1)** — 2026-05-07

A substantial release for the tidyverse's LLM interface package. The headline addition is OpenTelemetry instrumentation: when the `otel` package is present, every `$chat()`, `$stream()`, and tool call emits structured spans covering token usage, model metadata, and HTTP requests. The release also adds LM Studio support via `chat_lmstudio()`, improves streaming stability across Anthropic, OpenAI, and Snowflake providers, and introduces `stream_controller()` for programmatic cancellation of streaming responses — useful for Shiny apps needing a "Stop" button. A number of quieter fixes address token counting accuracy, tool metadata consistency, and edge cases in multi-turn conversations.

## LLM Tooling

**[datasette-llm 0.1a7](https://simonwillison.net/2026/May/5/datasette-llm/#atom-everything)** — date not provided

An alpha release of the plugin that brings LLM querying directly into Datasette. Still early-stage, but worth tracking if you use Datasette as a data exploration interface and want to layer in model-assisted queries.

**[llm-gemini 0.31](https://simonwillison.net/2026/May/7/llm-gemini/#atom-everything)** — date not provided

An update to Simon Willison's plugin connecting the `llm` CLI tool to Google's Gemini API. No dates in the frontmatter, but it ships alongside the broader `llm` ecosystem churn this week.

**[llm-echo 0.5a0](https://simonwillison.net/2026/May/5/llm-echo/#atom-everything)** — date not provided

A pre-release update to the echo plugin for the `llm` tool, primarily useful for testing pipelines that consume LLM output without burning tokens.

## Security & Hardening

**[Behind the Scenes: Hardening Firefox with Claude Mythos Preview](https://simonwillison.net/2026/May/7/firefox-claude-mythos/#atom-everything)** — date not provided

A look at Mozilla's use of Anthropic's Claude Mythos model in their Firefox security hardening pipeline. The piece covers how AI-assisted code review and fuzzing are being integrated into browser development, which is a meaningful real-world deployment rather than a demo.

**[datasette-referrer-policy 0.1](https://simonwillison.net/2026/May/5/datasette-referrer-policy/#atom-everything)** — date not provided

A small but sensible Datasette plugin that sets HTTP `Referrer-Policy` headers. Useful for anyone running a public-facing Datasette instance and wanting to avoid leaking URLs to third-party resources.

## Commentary

**[Vibe coding and agentic engineering are getting closer than I'd like](https://simonwillison.net/2026/May/6/vibe-coding-and-agentic-engineering/#atom-everything)** — date not provided

Willison expresses concern that the gap between casual AI-assisted coding and serious agentic engineering is narrowing faster than the tooling for oversight and auditability can keep up. Worth a read if you are thinking about where to draw lines in your own workflows.

**[Using Claude Code: The Unreasonable Effectiveness of HTML](https://simonwillison.net/2026/May/8/unreasonable-effectiveness-of-html/#atom-everything)** — date not provided

A practical field report on using Claude Code for web UI generation, arguing that plain HTML output is often more reliable and portable than framework-heavy alternatives. Dry and useful, less hype than the title suggests.
