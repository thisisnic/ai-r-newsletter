A busy week for the R LLM ecosystem, with ellmer hitting 0.4.0 and tidyllm landing a substantial 0.4.0 of its own.

---

## R Packages

**[ellmer 0.1.0 through 0.4.0](https://github.com/tidyverse/ellmer/blob/HEAD/NEWS.md#ellmer-0-1-0)** — 2026-04-13

The tidyverse's LLM interface package has accumulated several major releases, now landing at 0.4.0. The headline changes include a new `chat()` shorthand accepting strings like `chat("anthropic")`, `batch_chat()` and `parallel_chat()` graduating from experimental status, built-in web search and fetch tools for Claude, Gemini, and OpenAI, and a shift to OpenAI's responses endpoint. API keys are now retrieved via a `credentials` function rather than stored in the chat object, which closes an easy footgun around accidentally serializing secrets to disk.

**[tidyllm 0.4.0](https://github.com/edubruell/tidyllm/blob/HEAD/NEWS.md#tidyllm-0-4-0)** — 2026-03-17

A large release adding OpenRouter (300+ models) and llama.cpp as new providers, a `deep_research()` verb backed by Perplexity's sonar model, extended thinking support for Gemini and DeepSeek, and an `ellmer_tool()` bridge so tools defined in ellmer (including `btw` tools) work directly in tidyllm. The unified multi-turn tool loop now covers all major providers. A substantial release for users who want provider flexibility without rewriting their pipelines.

**[chattr 0.1.0–0.3.1](https://github.com/mlverse/chattr/blob/HEAD/NEWS.md#chattr-0-1-0)** — 2025-08-18

The RStudio-addin LLM chat package went through initial CRAN submission and several follow-up releases. The most consequential change is 0.3.0, which drops all direct provider integrations (OpenAI, Databricks, LlamaGPT) in favor of routing everything through ellmer, simplifying the dependency surface considerably.

**[ollamar 1.0.0–1.2.2](https://github.com/hauselin/ollama-r/blob/HEAD/NEWS.md#ollamar-1-0-0)** — 2025-03-24

The R client for local Ollama servers progressed from initial CRAN release through 1.2.2, adding structured output via the `format` parameter, tool calling with a `get_tool_calls()` helper, multi-image support, and a full set of message manipulation utilities. A reasonable choice if you want a thin, direct wrapper around the Ollama API rather than a multi-provider abstraction.

**[rollama 0.0.2–0.2.1](https://github.com/JBGruber/rollama/blob/HEAD/NEWS.md#rollama-0-0-2)** — 2026-03-23

Another Ollama client for R, with a focus on text annotation workflows. Recent versions added multi-model querying, dedicated embedding model support, structured output, custom headers for authentication, and an expanded annotation vignette. Somewhat overlapping with ollamar but oriented toward social science text analysis use cases.

## LLM Tooling

**[A Guide to Which AI to Use in the Agentic Era](https://www.oneusefulthing.org/p/a-guide-to-which-ai-to-use-in-the)** — 2026-02-18

Ethan Mollick's practical breakdown of how to think about model selection now that the relevant question is less "which chatbot" and more "which model for which agentic task." Covers reasoning models, coding assistants, and long-horizon agents as distinct categories with distinct tradeoffs.

**[Claude Code and What Comes Next](https://www.oneusefulthing.org/p/claude-code-and-what-comes-next)** — 2026-01-07

An assessment of Claude Code as an agentic coding tool, with observations on what it does well and where the current generation of coding agents still breaks down. The piece argues the gap between impressive demos and reliable production use is narrowing faster than most expect.

**[Claude Dispatch and the Power of Interfaces](https://www.oneusefulthing.org/p/claude-dispatch-and-the-power-of)** — 2026-03-31

Mollick's argument that the binding constraint on AI productivity is often the interface layer rather than the underlying model capability. Even capable models underperform when wrapped in clumsy tooling, a point relevant to anyone building R-based LLM workflows.

## Commentary

**[Against "Brain Damage"](https://www.oneusefulthing.org/p/against-brain-damage)** — 2025-07-07

A pushback against the recurring claim that using AI atrophies human cognitive skills. Mollick reviews the empirical evidence and argues the picture is more complicated: some tasks benefit from AI augmentation without measurable skill loss, while others warrant caution. Worth reading before citing the "AI makes you dumber" studies.

**[On Jagged AGI: o3, Gemini 2.5, and everything after](https://www.oneusefulthing.org/p/on-jagged-agi-o3-gemini-25-and-everything)** — 2025-04-20

An early piece on what it means when frontier models cross expert-level thresholds on some benchmarks while still failing straightforwardly on others. The "jagged frontier" framing remains useful for setting realistic expectations about what current models can and cannot be trusted with.

## Security

**[Cybersecurity Looks Like Proof of Work Now](https://simonwillison.net/2026/Apr/14/cybersecurity-proof-of-work/#atom-everything)**

Simon Willison's note on how AI-assisted attack generation is shifting the economics of security: the cost of producing plausible phishing and exploit attempts is dropping while the cost of verification stays roughly constant. Short but worth the two minutes.

**[Datasette PR #2689: Replace token-based CSRF with Sec-Fetch-Site header protection](https://simonwillison.net/2026/Apr/14/replace-token-based-csrf/#atom-everything)**

A practical security improvement in Datasette replacing traditional CSRF tokens with `Sec-Fetch-Site` header checks. Willison explains the reasoning; relevant to anyone building web-accessible data tools in the same vein.
