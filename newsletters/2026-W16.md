# AI Tooling Weekly

A week of model releases, security concerns, and the usual Datasette housekeeping.

## LLM Tooling and Model Updates

**[A Guide to Which AI to Use in the Agentic Era](https://www.oneusefulthing.org/p/a-guide-to-which-ai-to-use-in-the)**
The landscape has shifted enough that "which chatbot should I use" is no longer the right question. This piece maps out the current agent-capable models and where each actually earns its keep. Worth bookmarking as a reference rather than a one-time read.

**[ChatGPT Voice Mode is a Weaker Model](https://simonwillison.net/2026/Apr/10/voice-mode-is-weaker/#atom-everything)**
Simon Willison flags that ChatGPT's voice interface isn't just a different modality — it's running on a less capable model underneath. Useful to know if you've been relying on voice for anything that requires actual reasoning.

**[Meta's New Model is Muse Spark](https://simonwillison.net/2026/Apr/8/muse-spark/#atom-everything)**
Meta has a new model named Muse Spark, and the meta.ai chat interface has picked up some notable tooling. Willison gives a quick tour of what's actually interesting versus what's marketing.

**[Qwen3.6-35B-A3B on My Laptop Drew Me a Better Pelican Than Claude Opus 4.7](https://simonwillison.net/2026/Apr/16/qwen-beats-opus/#atom-everything)**
A local model running on consumer hardware outperforming a frontier cloud model on a specific task is the kind of data point worth noting. The gap between local and hosted continues to narrow in ways that matter for practitioners.

**[Gemini 3.1 Flash TTS](https://simonwillison.net/2026/Apr/15/gemini-flash-tts/#atom-everything)**
A brief look at Google's text-to-speech offering in the Gemini 3.1 Flash line. Willison covers what's actually available and how it compares to the current field.

**[Gemma 4 Audio with MLX](https://simonwillison.net/2026/Apr/12/mlx-audio/#atom-everything)**
Google's Gemma 4 gets audio capabilities via the MLX framework for Apple Silicon. For anyone running local models on Mac hardware, this extends what's practically available.

**[llm-anthropic 0.25](https://simonwillison.net/2026/Apr/16/llm-anthropic/#atom-everything)**
New release of Willison's Anthropic plugin for the `llm` CLI tool. If you're using `llm` in any pipelines, check the changelog.

## Datasette

**[datasette 1.0a27](https://simonwillison.net/2026/Apr/15/datasette/#atom-everything)** and **[1.0a28](https://simonwillison.net/2026/Apr/17/datasette/#atom-everything)**
Two alpha releases in quick succession as Datasette inches toward 1.0. The pace of releases suggests the finish line is getting close.

**[datasette PR #2689: Replace Token-Based CSRF with Sec-Fetch-Site Header Protection](https://simonwillison.net/2026/Apr/14/replace-token-based-csrf/#atom-everything)**
A notable security-oriented change in Datasette's internals, swapping traditional CSRF tokens for the `Sec-Fetch-Site` header approach. The PR notes explain the tradeoffs clearly and it's a pattern worth understanding for anyone building web applications.

**[datasette.io News Preview](https://simonwillison.net/2026/Apr/16/datasette-io-preview/#atom-everything)**
A preview of upcoming changes to the Datasette project site. Mostly relevant if you follow the project closely, but signals broader documentation and positioning work ahead of 1.0.

## Security

**[Cybersecurity Looks Like Proof of Work Now](https://simonwillison.net/2026/Apr/14/cybersecurity-proof-of-work/#atom-everything)**
A sharp observation that modern security has become less about clever defenses and more about sheer resource expenditure — both for attackers and defenders. The framing has implications for how AI changes the cost structure of attacks.

**[Anthropic's Project Glasswing — Restricting Claude Mythos to Security Researchers](https://simonwillison.net/2026/Apr/7/project-glasswing/#atom-everything)**
Anthropic is apparently limiting access to a more capable Claude variant to vetted security researchers. Willison finds this reasonable; the post is brief but the underlying policy question about frontier model access is worth watching.

**[Trusted Access for the Next Era of Cyber Defense](https://simonwillison.net/2026/Apr/14/trusted-access-openai/#atom-everything)**
OpenAI's framing of its role in cybersecurity. Willison links it without much editorial, which in context reads as mild skepticism. The document itself is worth reading for what it reveals about how labs are positioning with governments and enterprises.

## Commentary

**[Claude Dispatch and the Power of Interfaces](https://www.oneusefulthing.org/p/claude-dispatch-and-the-power-of)**
Even when the underlying model is capable, the interface constrains what actually gets done. This piece argues that tooling and workflow design matter as much as model capability — a practical point that gets underweighted in most model comparisons.

**[Against "Brain Damage"](https://www.oneusefulthing.org/p/against-brain-damage)**
A pushback on the reflexive claim that AI use atrophies thinking. The argument is nuanced: the risk is real in some contexts, nonexistent in others, and the framing usually obscures more than it reveals. Reasonable counterpoint to a recurring anxious take.

**[The Shape of the Thing](https://www.oneusefulthing.org/p/the-shape-of-the-thing)**
A state-of-play piece on where AI capability actually sits right now and what the near-term trajectory looks like. Less breathless than most, more useful for planning purposes.
