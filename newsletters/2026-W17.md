## LLM Tooling

**[llm 0.31](https://simonwillison.net/2026/Apr/24/llm/#atom-everything)** — date unavailable

A new release of Simon Willison's command-line LLM tool. Worth checking the changelog if you use this regularly, as point releases tend to carry useful quality-of-life changes.

**[llm-openrouter 0.6](https://simonwillison.net/2026/Apr/20/llm-openrouter/#atom-everything)** — date unavailable

Updated plugin for routing LLM requests through OpenRouter. Keeps the llm CLI connected to the expanding roster of models available via that gateway.

**[llm-openai-via-codex 0.1a0](https://simonwillison.net/2026/Apr/23/llm-openai-via-codex/#atom-everything)** — date unavailable

An alpha plugin that pipes requests through the Codex API endpoint as a back-channel to reach OpenAI models. Experimental, but paired with the pelican post below it serves as a practical demonstration of what the endpoint can do.

**[A pelican for GPT-5.5 via the semi-official Codex backdoor API](https://simonwillison.net/2026/Apr/23/gpt-5-5/#atom-everything)** — date unavailable

Willison uses the Codex API route to run GPT-5.5 and generate the obligatory pelican test image. Doubles as documentation of how to access the model before official availability through standard endpoints.

**[Claude Token Counter, now with model comparisons](https://simonwillison.net/2026/Apr/20/claude-token-counts/#atom-everything)** — date unavailable

A small but handy tool for counting tokens against Claude models, updated to show comparisons across model variants. Useful when you are trying to decide which model tier a given prompt fits into.

**[SQL functions in Google Sheets to fetch data from Datasette](https://simonwillison.net/2026/Apr/20/datasette-sql/#atom-everything)** — date unavailable

A practical integration that lets you call Datasette queries directly from Google Sheets using custom functions. Lowers the barrier for non-technical users to get at data sitting behind a Datasette instance.

**[Extract PDF text in your browser with LiteParse for the web](https://simonwillison.net/2026/Apr/23/liteparse-for-the-web/#atom-everything)** — date unavailable

Client-side PDF text extraction that runs entirely in the browser without sending data to a server. Relevant for any workflow where you want to feed document content into an LLM without a round-trip to a backend.
